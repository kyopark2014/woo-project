# 거래 조회 V2 (공통API) 테스트 케이스

## 테스트 개요
- **테스트 대상**: 거래 조회 V2 API
- **End Point**: /common-api/v2/customers/{customerNo}/tx-histories
- **기능**: 거래처번호에 따른 거래 내역 조회
- **작업사항**: 정산기준조정금액에 '배민 부담 가게배달팁' 코드(BARO_WOOWABROS_CHARGE_DELIVERY_TIP) 추가

## 테스트 케이스

### 테스트 케이스 1: 배민 부담 가게배달팁이 없는 경우 API 응답 확인
- **목적**: 배민 부담 가게배달팁이 없는 거래에서 API가 정상적으로 응답하는지 확인
- **선행 조건**: 
  - 배민 부담 가게배달팁이 없는 거래 데이터가 존재함
  - 거래처번호가 유효함
- **테스트 단계**:
  1. API 호출: GET /common-api/v2/customers/{customerNo}/tx-histories?txDate=2024-XX-XX
  2. 응답 확인
- **예상 결과**:
  - HTTP 상태 코드: 200 OK
  - content[].adjustAmounts 배열이 비어있거나, BARO_WOOWABROS_CHARGE_DELIVERY_TIP 코드가 없음
  - content[].transactionAmounts에 ORDER_AMOUNT(주문금액)와 BARO_DELIVERYTIP(가게배달팁) 항목이 정상적으로 포함됨
- **실제 결과**: (테스트 후 기록)

  - BARO_WOOWABROS_CHARGE_DELIVERY_TIP 항목의 supplyAmount 값이 2,727원

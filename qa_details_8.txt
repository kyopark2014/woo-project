# 테스트 케이스: 부가세 참고자료 매출 상세 메일 및 메일/팩스 발송 API

## 1. 부가세 참고자료 매출 상세 메일 테스트 케이스

### TC-01: 기본 매출 상세 메일 발송 기능 검증
**설명**: 부가세 참고자료 매출 상세 메일 발송 기능의 기본 동작을 검증
**선행 조건**: 
- 유효한 거래처번호(customerNo)가 존재함
- 해당 거래처에 대한 매출 데이터가 존재함

**테스트 단계**:
1. 부가세 참고자료 매출 상세 메일 API 호출
2. 필수 파라미터(거래처번호, 시작 연월, 종료 연월, 이메일 주소)를 입력
3. API 응답 확인

**예상 결과**:
- 응답 코드 200 OK 반환
- 메일이 지정된 이메일 주소로 성공적으로 발송됨
- 발송된 메일에 매출 상세 내역이 포함됨

### TC-02: 배민 부담 가게배달팁이 포함된 매출 상세 메일 발송 검증
**설명**: 배민 부담 가게배달팁(HVA 배달팁 할인)이 포함된 매출 상세 메일 발송 기능 검증
**선행 조건**: 
- 유효한 거래처번호가 존재함
- 해당 거래처에 배민 부담 가게배달팁 거래 데이터가 존재함

**테스트 단계**:
1. 부가세 참고자료 매출 상세 메일 API 호출
2. 필수 파라미터 입력 및 배민 부담 가게배달팁이 포함된 기간 지정
3. API 응답 확인
4. 수신된 메일 내용 확인

**예상 결과**:
- 응답 코드 200 OK 반환
- 메일이 성공적으로 발송됨
- 발송된 메일의 매출 상세 내역에 '배민 부담 가게배달팁' 항목이 포함됨
- 해당 항목이 '기타매출' 구분으로 올바르게 표시됨
- 공급가액, 부가세, 합계 금액이 정확히 표시됨

### TC-03: 매출 상세 메일 발송 시 필수 파라미터 누락 검증
**설명**: 필수 파라미터가 누락된 경우 API 동작 검증
**테스트 단계**:
1. 부가세 참고자료 매출 상세 메일 API 호출 시 필수 파라미터 누락
   - 케이스 1: 거래처번호 누락
   - 케이스 2: 시작 연월 누락
   - 케이스 3: 종료 연월 누락
   - 케이스 4: 이메일 주소 누락
2. 각 케이스별 API 응답 확인

**예상 결과**:
- 각 케이스에서 적절한 오류 코드와 메시지 반환
- 메일이 발송되지 않음

### TC-04: 매출 상세 메일 발송 시 데이터 없는 경우 검증
**설명**: 지정된 기간에 매출 데이터가 없는 경우 API 동작 검증
**선행 조건**: 
- 유효한 거래처번호가 존재함
- 지정된 기간에 매출 데이터가 없음

**테스트 단계**:
1. 부가세 참고자료 매출 상세 메일 API 호출
2. 매출 데이터가 없는 기간 지정
3. API 응답 확인

**예상 결과**:
- 응답 코드 200 OK 반환
- 메일이 발송됨
- 메일 내용에 데이터가 없음을 나타내는 메시지 포함

이제 "부가세 참고자료 FAX 전송 V3" 및 "메일/팩스 발송 API"에 관한 테스트 케이스를 작성하겠습니다. 문서에서 관련 정보를 확인한 결과, 이 API들은 부가세 참고자료를 FAX나 메일로 전송하는 기능을 담당하고 있으며, 특히 "배민 부담 가게배달팁"(HVA 배달팁 할인)이 기타매출로 포함되어 제공되는 것을 확인했습니다.

# 부가세 참고자료 FAX 전송 V3 테스트 케이스

## TC-01: 기본 FAX 전송 기능 검증
**설명**: 부가세 참고자료 FAX 전송 API의 기본 기능 검증
**선행조건**: 유효한 거래처번호와 FAX 번호가 존재함
**테스트 단계**:
1. 필수 파라미터(거래처번호, 시작 연월, 종료 연월, FAX 번호)를 포함하여 API 호출
2. 응답 코드와 메시지 확인

**예상 결과**:
- 상태 코드 200 반환
- 성공 메시지 확인
- FAX 전송 요청이 정상적으로 처리됨

## TC-02: 배민 부담 가게배달팁이 포함된 부가세 참고자료 FAX 전송
**설명**: HVA 배달팁 할인(배민 부담 가게배달팁)이 기타매출로 포함되어 FAX로 전송되는지 검증
**선행조건**: 
- 유효한 거래처번호와 FAX 번호가 존재함
- 해당 거래처에 배민 부담 가게배달팁 거래 내역이 존재함

**테스트 단계**:
1. 배민 부담 가게배달팁 거래가 있는 기간의 시작 연월과 종료 연월을 설정
2. 필수 파라미터와 함께 API 호출
3. 전송된 FAX 문서 내용 확인

**예상 결과**:
- 상태 코드 200 반환
- 전송된 FAX 문서에 기타매출 항목에 배민 부담 가게배달팁 금액이 포함됨
- 기타매출 항목의 공급가액, 부가세, 합계가 정확히 표시됨

## TC-03: 만나서결제 제외 옵션을 적용한 FAX 전송
**설명**: 만나서결제를 제외한 부가세 참고자료 FAX 전송 검증
**선행조건**: 유효한 거래처번호와 FAX 번호가 존재함
**테스트 단계**:
1. excludeMeet 파라미터를 true로 설정하여 API 호출
2. 전송된 FAX 문서 내용 확인

**예상 결과**:
- 상태 코드 200 반환
- 전송된 FAX 문서에 만나서결제 관련 항목이 제외됨
- 배민 부담 가게배달팁은 여전히 기타매출에 포함됨

## TC-04: 잘못된 파라미터로 FAX 전송 시도
**설명**: 잘못된 파라미터로 API 호출 시 적절한 오류 처리 검증
**테스트 단계**:
1. 존재하지 않는 거래처번호로 API 호출
2. 잘못된 형식의 FAX 번호로 API 호출
3. 잘못된 형식의 시작/종료 연월로 API 호출

**예상 결과**:
- 적절한 오류 코드와 메시지 반환
- 각 오류 상황에 맞는 구체적인 오류 메시지 제공

## TC-05: 대용량 데이터에 대한 FAX 전송 성능
**설명**: 많은 거래 내역이 있는 경우 FAX 전송 성능 검증
**선행조건**: 대량의 거래 내역이 있는 거래처번호와 유효한 FAX 번호가 존재함
**테스트 단계**:
1. 많은 거래 내역이 있는 기간의 시작 연월과 종료 연월을 설정
2. API 호출 및 응답 시간 측정
3. FAX 전송 완료 여부 확인

**예상 결과**:
- 적절한 시간 내에 응답 반환
- FAX 전송이 성공적으로 완료됨
- 대용량 데이터가 올바르게 처리됨

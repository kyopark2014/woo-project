# 테스트 케이스: 정산지급내역 거래건별 조회 V3 - 배민 부담 가게배달팁 필드 추가

## 1. 기본 정보
- **테스트 항목**: 정산지급내역 거래건별 조회 V3
- **End Point**: 명시되지 않음 (문서에서는 `/api/v3/customer/{customerNo}/give-history/detail` 형태로 추정됨)
- **Request Parameters**: startDate, endDate, customerNo

## 2. 테스트 목적
- 정산지급내역 거래건별 조회 API V3에 배민 부담 가게배달팁 필드가 정상적으로 추가되었는지 확인
- 배민 부담 가게배달팁 값이 정상적으로 조회되는지 확인

## 3. 선행 조건
- 테스트 환경에 접근 권한이 있어야 함
- 테스트용 거래처번호(customerNo)가 준비되어 있어야 함
- 배민 부담 가게배달팁이 적용된 주문 데이터가 존재해야 함

## 4. 테스트 케이스

### 테스트 케이스 1: 배민 부담 가게배달팁이 없는 주문 조회
- **입력 데이터**:
  - startDate: 배민 부담 가게배달팁이 없는 주문이 있는 날짜
  - endDate: 배민 부담 가게배달팁이 없는 주문이 있는 날짜
  - customerNo: 테스트용 거래처번호
- **예상 결과**:
  - API 응답에 `baroWoowabrosChargeDelivertyTip` 필드가 포함됨
  - `baroWoowabrosChargeDelivertyTip` 값이 0으로 표시됨
  - `txBaroBaeminDeliveryTipAmount` 값이 실제 배달팁 금액(예: 3,000원)으로 표시됨
  - 정산기준금액(baseAmount)이 주문금액과 배달팁의 합계(예: 13,000원)로 표시됨

### 테스트 케이스 2: 배민 부담 가게배달팁이 있는 주문 조회
- **입력 데이터**:
  - startDate: 배민 부담 가게배달팁이 있는 주문이 있는 날짜
  - endDate: 배민 부담 가게배달팁이 있는 주문이 있는 날짜
  - customerNo: 테스트용 거래처번호
- **예상 결과**:
  - API 응답에 `baroWoowabrosChargeDelivertyTip` 필드가 포함됨
  - `baroWoowabrosChargeDelivertyTip` 값이 배민 부담 가게배달팁 금액(예: 3,000원)으로 표시됨
  - `txBaroBaeminDeliveryTipAmount` 값이 0으로 표시됨
  - 정산기준금액(baseAmount)이 주문금액과 배민 부담 가게배달팁의 합계(예: 13,000원)로 표시됨

### 테스트 케이스 3: 여러 주문 데이터 조회 시 배민 부담 가게배달팁 필드 확인
- **입력 데이터**:
  - startDate: 테스트 기간 시작일
  - endDate: 테스트 기간 종료일 (여러 주문 데이터가 포함된 기간)
  - customerNo: 테스트용 거래처번호
- **예상 결과**:
  - 모든 주문 데이터에 `baroWoowabrosChargeDelivertyTip` 필드가 포함됨
  - 배민 부담 가게배달팁이 적용된 주문은 해당 값이 표시되고, 적용되지 않은 주문은 0으로 표시됨
  - 정산기준금액(baseAmount)이 각 주문에 맞게 정확히 계산됨

### 테스트 케이스 4: 부분 취소된 주문에서 배민 부담 가게배달팁 필드 확인
- **입력 데이터**:
  - startDate: 부분 취소된 주문이 있는 날짜
  - endDate: 부분 취소된 주문이 있는 날짜
  - customerNo: 테스트용 거래처번호
- **예상 결과**:
  - 부분 취소된 주문에도 `baroWoowabrosChargeDelivertyTip` 필드가 포함됨
  - 부분 취소에 따른 배민 부담 가게배달팁 금액이 정확히 반영됨
  - 정산기준금액(baseAmount)이 부분 취소를 반영하여 정확히 계산됨

### 테스트 케이스 5: 날짜 범위를 넓게 설정하여 조회
- **입력 데이터**:
  - startDate: 1개월 전 날짜
  - endDate: 현재 날짜
  - customerNo: 테스트용 거래처번호
- **예상 결과**:
  - 모든 조회된 데이터에 `baroWoowabrosChargeDelivertyTip` 필드가 포함됨
  - 기간 내 모든 주문에 대해 배민 부담 가게배달팁 정보가 정확히 표시됨
  - 페이지네이션이 적용된 경우 모든 페이지에서 일관되게 필드가 표시됨

## 5. 검증 방법
1. API 응답에 `baroWoowabrosChargeDelivertyTip` 필드가 존재하는지 확인
2. 배민 부담 가게배달팁이 있는 주문과 없는 주문에서 필드 값이 정확히 표시되는지 확인
3. 정산기준금액(baseAmount)이 주문금액, 배달팁, 배민 부담 가게배달팁을 정확히 반영하여 계산되는지 확인
4. 다양한 조건(날짜 범위, 여러 주문, 부분 취소 등)에서 일관되게 동작하는지 확인

## 6. 예상 결과
- 모든 API 응답에 `baroWoowabrosChargeDelivertyTip` 필드가 추가됨
- 배민 부담 가게배달팁이 적용된 주문에서는 해당 값이 정확히 표시됨
- 배민 부담 가게배달팁이 적용되지 않은 주문에서는 값이 0으로 표시됨
- 정산기준금액과 정산예정금액이 배민 부담 가게배달팁을 포함하여 정확히 계산됨

이 테스트 케이스를 통해 정산지급내역 거래건별 조회 V3 API에 배민 부담 가게배달팁 필드가 정상적으로 추가되었는지, 그리고 해당 필드가 다양한 상황에서 정확한 값을 반환하는지 검증할 수 있습니다.

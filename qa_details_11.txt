# 정산지급내역 거래건별 조회 V3 테스트 케이스

## 1. 기본 정보
- **API 명**: 정산지급내역 거래건별 조회 V3
- **Request Parameters**: 
  - startDate (필수): 지급예정일자(시작일) - yyyy-MM-dd 형식
  - endDate (필수): 지급예정일자(종료일) - yyyy-MM-dd 형식
  - customerNo (필수): 거래처번호 - 숫자 12자리

## 2. 테스트 케이스

### TC-01: 정상 조회 - 할인 없는 경우
**목적**: 할인이 없는 정산지급내역을 정상적으로 조회할 수 있는지 확인
- **입력값**:
  - startDate: 현재 날짜로부터 30일 전
  - endDate: 현재 날짜
  - customerNo: 유효한 거래처번호
- **예상 결과**:
  - HTTP 상태 코드: 200 OK
  - 응답 데이터에 txBaroBaeminDeliveryTipAmount(배달팁)가 3,000원으로 표시됨
  - 응답 데이터에 baroWoowabrosChargeDelivertyTip(배민 부담 가게배달팁)이 0원으로 표시됨
  - 응답 데이터에 baseAmount(정산기준금액)가 13,000원으로 표시됨

### TC-02: 정상 조회 - 배민 부담 가게배달팁이 있는 경우
**목적**: 배민 부담 가게배달팁이 있는 정산지급내역을 정상적으로 조회할 수 있는지 확인
- **입력값**:
  - startDate: 현재 날짜로부터 30일 전
  - endDate: 현재 날짜
  - customerNo: 배민 부담 가게배달팁이 있는 거래처번호
- **예상 결과**:
  - HTTP 상태 코드: 200 OK
  - 응답 데이터에 txBaroBaeminDeliveryTipAmount(배달팁)가 0원으로 표시됨
  - 응답 데이터에 baroWoowabrosChargeDelivertyTip(배민 부담 가게배달팁)이 3,000원으로 표시됨
  - 응답 데이터에 baseAmount(정산기준금액)가 13,000원으로 표시됨

### TC-03: 유효하지 않은 날짜 형식으로 조회
**목적**: 유효하지 않은 날짜 형식으로 요청 시 적절한 오류 응답을 반환하는지 확인
- **입력값**:
  - startDate: "2023/01/01" (잘못된 형식)
  - endDate: 현재 날짜
  - customerNo: 유효한 거래처번호
- **예상 결과**:
  - HTTP 상태 코드: 400 Bad Request
  - 오류 메시지: 날짜 형식 오류 관련 메시지

### TC-04: 시작일이 종료일보다 늦은 경우
**목적**: 시작일이 종료일보다 늦은 경우 적절한 오류 응답을 반환하는지 확인
- **입력값**:
  - startDate: 현재 날짜
  - endDate: 현재 날짜로부터 30일 전
  - customerNo: 유효한 거래처번호
- **예상 결과**:
  - HTTP 상태 코드: 400 Bad Request
  - 오류 메시지: 날짜 범위 오류 관련 메시지

### TC-05: 존재하지 않는 거래처번호로 조회
**목적**: 존재하지 않는 거래처번호로 요청 시 적절한 응답을 반환하는지 확인
- **입력값**:
  - startDate: 현재 날짜로부터 30일 전
  - endDate: 현재 날짜
  - customerNo: 존재하지 않는 거래처번호
- **예상 결과**:
  - HTTP 상태 코드: 404 Not Found 또는 빈 결과 목록

### TC-06: 조회 기간이 너무 긴 경우
**목적**: 조회 기간이 시스템 제한을 초과하는 경우 적절한 오류 응답을 반환하는지 확인
- **입력값**:
  - startDate: 현재 날짜로부터 1년 전
  - endDate: 현재 날짜
  - customerNo: 유효한 거래처번호
- **예상 결과**:
  - HTTP 상태 코드: 400 Bad Request
  - 오류 메시지: 조회 기간 제한 관련 메시지

### TC-07: 거래처번호 형식 오류
**목적**: 거래처번호 형식이 잘못된 경우 적절한 오류 응답을 반환하는지 확인
- **입력값**:
  - startDate: 현재 날짜로부터 30일 전
  - endDate: 현재 날짜
  - customerNo: "ABC123" (잘못된 형식)
- **예상 결과**:
  - HTTP 상태 코드: 400 Bad Request
  - 오류 메시지: 거래처번호 형식 오류 관련 메시지

### TC-08: 부분 취소된 거래 조회
**목적**: 부분 취소된 거래의 정산지급내역을 정상적으로 조회할 수 있는지 확인
- **입력값**:
  - startDate: 현재 날짜로부터 30일 전
  - endDate: 현재 날짜
  - customerNo: 부분 취소 거래가 있는 거래처번호
- **예상 결과**:
  - HTTP 상태 코드: 200 OK
  - 응답 데이터에 부분 취소된 거래 정보가 포함됨
  - billingCancelTradeNo 필드에 부분 취소 번호가 표시됨

## 3. 추가 검증 사항
- 응답 데이터의 모든 필수 필드가 존재하는지 확인
- 금액 관련 필드의 값이 예상 범위 내에 있는지 확인
- 날짜 형식이 올바른지 확인
- 응답 시간이 적절한지 확인 (성능 테스트)

이 테스트 케이스들은 "정산지급내역 거래건별 조회 V3" API의 다양한 시나리오를 검증하며, 특히 배민 부담 가게배달팁 관련 필드가 올바르게 표시되는지 확인합니다.
